# Claude Code Integration

Claude Code operates as the **orchestrator** in a multi-provider agent architecture. This document covers the full integration: configuration sync, hook pipeline, slash commands, sub-agents, and the policy contract system.

## Architecture

```
Claude Code (orchestrator)
  ├─ Gemini CLI (researcher/critic) — async FIFO handshake
  ├─ Codex (logic verifier) — log-only reasoning traces
  └─ cli-proxy-api (localhost:4001) — unified auth + cost logging + routing
```

Configuration is managed declaratively through Nix:
- **Static files** (commands, agents, skills, hooks) are symlinked via `home.file`
- **Dynamic settings** (permissions, MCP servers, hooks) are deep-merged into `~/.claude.json` and `~/.claude/settings.json` via activation scripts
- **Policy-generated hooks** are produced by `lib/agent-policy/` at build time and merged with base hooks

## Configuration Sync

When you run `just apply`, home-manager:

1. Symlinks `dotfiles/claude/{commands,agents,skills,hooks}` to `~/.claude/`
2. Generates MCP server config from `modules/agents/agents-mcp.nix` (SSoT) and merges into `~/.claude.json`
3. Generates settings from `dotfiles/claude/settings.json` + policy hooks and merges into `~/.claude/settings.json`
4. Preserves runtime data (projects, tips, OAuth tokens) through `jq` deep-merge

Key implementation files:
- `lib/sync-mutable-config.nix` — `mkJsonSync` (deep-merge) and `mkFileCopy` (overwrite with backup)
- `lib/mcp-adapters.nix` — transforms canonical MCP definitions to Claude/Gemini/Codex formats

## Hook Pipeline

All hooks are shell scripts that read JSON from stdin and return exit codes: 0 (allow), 1 (retry), 2 (block).

### UserPromptSubmit

| Hook | Purpose |
|---|---|
| `complexity-router.sh` | Injects S/M/L classification prompt on first turn |

### PreToolUse

| Matcher | Hook | Purpose |
|---|---|---|
| `Bash` | `cmd-guard.sh` | Blocks destructive shell commands |
| `Bash` | `branch-guard.sh` | Protects main/production branches |
| `Write\|Edit\|Read` | `path-guard.sh` | Blocks `.env`, private keys, credentials |
| `Write\|Edit\|Read` | `complexity-gate.sh` | Blocks L-complexity without approval |
| `Write\|Edit\|NotebookEdit` | `phase-gate-claude.sh` * | Contract-generated phase enforcement |
| `Write\|Edit\|NotebookEdit` | `strategy-lint-claude.sh` * | Contract-generated strategy validation |
| `Write\|Edit\|Read` | `path-guard-claude.sh` * | Contract-generated from `global.sensitivePatterns` |
| `Agent` | `cost-gate.sh` | Blocks at >80% context window |

### PostToolUse

| Matcher | Hook | Purpose |
|---|---|---|
| `Bash` | `proxy-route.sh` | Logs tool usage to cli-proxy-api |
| `Bash` | `escalation-gate.sh` | Tracks failures, auto-stash at 3rd, escalates |
| `Bash` | `test-feedback.sh` | Parses test output, extracts top errors |
| `Write\|Edit` | `escalation-gate.sh` | Same failure tracking for file mutations |
| `Write\|Edit` | `auto-lint.sh` | Auto-format after writes |
| `Write\|Edit\|NotebookEdit` | `live-oracle-claude.sh` * | Contract-generated runtime health checks |
| all | `reasoning-trace-claude.sh` * | Contract-generated reasoning separation |

### Stop

| Hook | Purpose |
|---|---|
| `agent-notify.sh claude` | macOS notification on session end |

*\* Generated by Agent Policy Contract system at `nix build` time*

## Agent Policy Contract

The policy system (`lib/agent-policy/`) uses Nix modules as a DDD-style IoC container. Claude's contract implementation enables:

- **Silent reasoning** — chain-of-thought logged to `/tmp/agent-traces/claude/`, only decisions shown
- **Phase gate** — Write/Edit blocked until L-complexity strategy is approved
- **Strategy lint** — requires `pre-mortem`, `tradeoffs`, `peer-review` sections; gates on Gemini review
- **Live oracle** — `nix flake check --no-build` runs after every file mutation
- **Path guard** — sensitive file patterns from `agentPolicy.global.sensitivePatterns`

Full documentation: [agent-policy-contract.md](agent-policy-contract.md)

## Modifying Configuration

### Add/Remove Permissions

Edit `dotfiles/claude/settings.json`:
```json
{
  "permissions": {
    "allow": ["Read(*)", "Glob(*)", "YourNewTool(*)"]
  }
}
```

Then apply: `just apply`

### Add MCP Servers

Edit `modules/agents/agents-mcp.nix` (single source of truth):
```nix
programs.mcp.servers.my-server = {
  command = "npx";
  args = ["-y" "@example/mcp-server@latest"];
};
```

The adapter (`lib/mcp-adapters.nix`) automatically converts to Claude/Gemini/Codex formats.

### Add a Hook

For one-off hooks, add to `dotfiles/claude/settings.json`. For policy-driven hooks, create or modify a mixin in `lib/agent-policy/mixins/` — it will be generated and merged at build time.
